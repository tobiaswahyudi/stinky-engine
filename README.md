# STINKY GAME ENGINE
![the](https://img.shields.io/badge/the-848484) ![stinky](https://img.shields.io/badge/STINKY-5eb110) ![game](https://img.shields.io/badge/game-848484) ![engine](https://img.shields.io/badge/engine-848484) 
## What is it?
Fully canvas-based game engine.
No library dependencies, except PeerJS and its deps if you want networking.

## Features
- Web-native, low overhead cross-platform for most features.
- Easy to deploy, free with tools like github pages or similar.
- Use browser tools to debug your game.
- Deploy exactly the code you write.
- Super VCS friendly, 100% source code with no binaries.
- Easy to replace expensive modules like collision with WASM if needed.

## What is the point?

- 2D Canvas has an incredibly strong, underrated API.
- Fun to work with, in my opinion.
- Web games has the same vibe as flash games and more power via browser APIs and resources (networking, WebGPU, WASM, service workers, ...).
- ***Most importantly:*** I just wanted to write a "game engine" and implement most of its internals, like collision and rollback-based multiplayer.
